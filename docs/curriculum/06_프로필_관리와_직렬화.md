## 1. serde 직렬화/역직렬화

### 개념

```
Serialize (직렬화)                 Deserialize (역직렬화)
Rust struct → JSON 문자열          JSON 문자열 → Rust struct
────────────────────              ────────────────────
MonitorProfile {                   {"schema_version":"0.2.0",
  name: "home",                     "name":"home",
  width: 1920,         ───▶         "width":1920,
  ...                               ...}
}
                                         │
                                         ▼
                                   MonitorProfile {
                                     name: "home",
                         ◀───         width: 1920,
                                     ...
                                   }
```

### DMaster에서의 사용

**Cargo.toml 의존성:**
```toml
serde = { version = "1.0", features = ["derive"] }   # derive 매크로 활성화
serde_json = "1.0"
```

**struct에 derive 추가:**
```rust
#[derive(Serialize, Deserialize)]
pub struct MonitorProfile {
    pub schema_version: String,
    pub name: String,
    pub description: Option<String>,   // JSON: null 또는 "설명"
    pub created_at: String,
    pub topology: DisplayTopology,     // JSON: "Extend" 등
    pub displays: Vec<DisplayConfig>,  // JSON: 배열
}
```

**직렬화 (struct → JSON):**
```rust
let json = serde_json::to_string_pretty(&stored_profile)
    .map_err(|error| format!("failed to serialize: {error}"))?;
// json = "{\n  \"schema_version\": \"0.2.0\",\n  ..."
```

**역직렬화 (JSON → struct):**
```rust
let profile: MonitorProfile = serde_json::from_str(&content)
    .map_err(|error| format!("failed to parse: {error}"))?;
```

### serde의 자동 매핑 규칙

| Rust 타입 | JSON 타입 | 예시 |
|-----------|-----------|------|
| `String` | string | `"home"` |
| `u32`, `i32` | number | `1920`, `-100` |
| `bool` | boolean | `true` |
| `Option<T>` | null 또는 값 | `null`, `"description"` |
| `Vec<T>` | array | `[{...}, {...}]` |
| `enum` (unit variant) | string | `"Extend"` |
| `enum(i32)` (data variant) | object | `{"Unknown": 42}` |

**Enum 직렬화 예시:**
```rust
// Rust
DisplayTopology::Extend        →  JSON: "Extend"
DisplayTopology::Unknown(42)   →  JSON: {"Unknown": 42}
```

### #[serde] 속성

```rust
#[derive(Deserialize)]
struct QueryDisplay {
    connector: String,
    #[serde(default)]           // JSON에 이 필드가 없으면 기본값 사용
    available_modes: Vec<AvailableMode>,  // 기본값: 빈 Vec
}
```

## 2. 파일 I/O (std::fs)

### 프로필 디렉토리 결정

```rust
pub fn profiles_dir() -> Result<PathBuf, String> {
    let home = std::env::var_os("HOME")           // Linux: /home/user
        .or_else(|| std::env::var_os("USERPROFILE"))  // Windows: C:\Users\user
        .map(PathBuf::from)
        .ok_or_else(|| String::from("HOME/USERPROFILE is not set"))?;

    Ok(home.join(".dmaster").join("profiles"))
    // Linux:   /home/user/.dmaster/profiles
    // Windows: C:\Users\user\.dmaster\profiles
}
```

**메서드 체이닝 분해:**

```
env::var_os("HOME")                    → Option<OsString>
  .or_else(|| var_os("USERPROFILE"))   → Option<OsString>  (HOME 없으면 USERPROFILE 시도)
  .map(PathBuf::from)                  → Option<PathBuf>   (OsString → PathBuf 변환)
  .ok_or_else(|| String::from("..."))  → Result<PathBuf, String> (None → Err 변환)
```

### 저장 (save_profile)

```rust
pub fn save_profile(
    name: &str,
    description: Option<String>,
    profile: &DisplayProfile,
) -> Result<PathBuf, String> {
    let dir = profiles_dir()?;

    // 1. 디렉토리 생성 (이미 있으면 무시)
    fs::create_dir_all(&dir).map_err(|error| {
        format!("failed to create directory '{}': {error}", dir.display())
    })?;

    // 2. MonitorProfile 구성
    let stored_profile = MonitorProfile::new(
        name.to_string(),
        description,
        current_timestamp_string(),
        profile.clone(),
    );

    // 3. 파일명 생성 (안전한 문자만)
    let file_path = dir.join(format!("{}.json", sanitize_profile_name(name)));

    // 4. JSON 직렬화
    let json = serde_json::to_string_pretty(&stored_profile)
        .map_err(|error| format!("failed to serialize: {error}"))?;

    // 5. 파일 쓰기
    fs::write(&file_path, json)
        .map_err(|error| format!("failed to write: {error}"))?;

    Ok(file_path)
}
```

### 로드 (load_profiles)

```rust
pub fn load_profiles() -> Result<Vec<ProfileInfo>, String> {
    let dir = profiles_dir()?;
    if !dir.exists() {
        return Ok(Vec::new());    // 디렉토리 없으면 빈 목록
    }

    let entries = fs::read_dir(&dir).map_err(|e| format!("..."))?;
    let mut profiles = Vec::new();

    for entry in entries {
        let entry = entry.map_err(|e| format!("..."))?;
        let path = entry.path();

        if !path.is_file() { continue; }    // 디렉토리는 건너뜀

        let content = fs::read_to_string(&path).map_err(|e| format!("..."))?;
        let profile = parse_profile_content(&path, &content)?;
        profiles.push(ProfileInfo {
            name: profile.name.clone(),
            path: path.to_string_lossy().to_string(),
            profile,
        });
    }

    profiles.sort_by(|a, b| a.name.cmp(&b.name));   // 이름순 정렬
    Ok(profiles)
}
```

### 파일명 안전화 (sanitize_profile_name)

```rust
fn sanitize_profile_name(name: &str) -> String {
    let sanitized = name.chars()
        .map(|ch| match ch {
            'a'..='z' | 'A'..='Z' | '0'..='9' | '-' | '_' => ch,
            _ => '_',       // 한글, 공백, 특수문자 → '_'
        })
        .collect::<String>();

    if sanitized.is_empty() {
        String::from("profile")
    } else {
        sanitized
    }
}
```

```
"내 모니터 설정!" → "________!"  → "__________"
"home-dual"      → "home-dual"
""               → "profile"
```

## 3. Error Handling 체이닝

### DMaster의 에러 처리 전략

```
┌─────────────────────────────────────────────┐
│ DMaster의 규칙:                               │
│                                               │
│ 1. 모든 public 함수는 Result<T, String> 반환   │
│ 2. 에러 메시지에 "뭐가 실패했는지" + "원인"을 포함 │
│ 3. map_err로 하위 에러를 상위 문맥에 맞게 변환    │
│ 4. ?로 에러를 즉시 전파                          │
└─────────────────────────────────────────────┘
```

### map_err 체이닝 패턴

```rust
// 단일 연산: IO 에러를 설명 문자열로 변환
fs::create_dir_all(&dir)
    .map_err(|error| format!("failed to create directory '{}': {error}", dir.display()))?;
```

**에러 변환 흐름:**
```
fs::create_dir_all(&dir)
    → Ok(())                    → 성공, 계속 진행
    → Err(io::Error)            
        → map_err 적용           
        → Err("failed to create directory '/home/user/.dmaster/profiles': Permission denied")
        → ? 적용
        → return Err("failed to create ...")
```

### 연쇄 에러 처리 (and_then 패턴)

```rust
// gui/src/main.rs
let result = get_display_profile().and_then(|profile| {
    save_profile(name.as_str(), desc_opt, &profile)
        .map(|path| path.display().to_string())
});
```

**흐름:**
```
get_display_profile()
    → Ok(profile) ──────▶ save_profile(name, desc, &profile)
                               → Ok(path) ──────▶ path.display().to_string()
                               → Err("save failed") ──▶ return Err("save failed")
    → Err("query failed") ──▶ return Err("query failed")
```

`and_then`: `Ok`면 다음 연산 실행, `Err`면 그대로 전파. 에러가 발생한 단계에서 적절한 메시지가 포함됨.

## 4. 레거시 포맷 마이그레이션

### 두 가지 프로필 포맷

```rust
const LEGACY_PROFILE_HEADER: &str = "DMaster_v0_1_1";

fn parse_profile_content(path: &Path, content: &str) -> Result<MonitorProfile, String> {
    // 첫 줄이 레거시 헤더인지 확인
    if content.lines().next() == Some(LEGACY_PROFILE_HEADER) {
        return parse_legacy_profile(path, content);
    }

    // 아니면 현재 JSON 포맷으로 파싱
    serde_json::from_str::<MonitorProfile>(content)
        .map_err(|error| format!("failed to parse: {error}"))
}
```

**레거시 포맷:**
```
DMaster_v0_1_1          ← 헤더 (첫 줄)
{"topology":"Extend",   ← 나머지는 DisplayProfile JSON
 "displays":[...]}
```

**현재 포맷:**
```json
{
  "schema_version": "0.2.0",
  "name": "home_dual",
  "description": "집 듀얼 설정",
  "created_at": "1709900000",
  "topology": "Extend",
  "displays": [...]
}
```

### 레거시 파싱

```rust
fn parse_legacy_profile(path: &Path, content: &str) -> Result<MonitorProfile, String> {
    let mut lines = content.lines();
    let _header = lines.next();   // 첫 줄(헤더) 건너뜀
    let json_data = lines.collect::<Vec<&str>>().join("\n");

    // 나머지를 DisplayProfile로 파싱
    let legacy_profile = serde_json::from_str::<DisplayProfile>(&json_data)?;

    // MonitorProfile로 래핑 (메타데이터 추가)
    Ok(MonitorProfile::new(
        path.file_stem()                     // "home_dual.json" → "home_dual"
            .unwrap_or_default()
            .to_string_lossy()
            .to_string(),
        Some(String::from("Imported from legacy DMaster profile format")),
        current_timestamp_string(),
        legacy_profile,
    ))
}
```

📌 **하위 호환성 패턴**: 새 코드가 구 포맷도 읽을 수 있다. 로드 시 자동 변환하되, 파일 자체는 수정하지 않음.

## 5. 타임스탬프 처리

```rust
fn current_timestamp_string() -> String {
    match SystemTime::now().duration_since(UNIX_EPOCH) {
        Ok(duration) => duration.as_secs().to_string(),   // "1709900000"
        Err(_) => String::from("0"),                       // 시스템 시계 이상 시
    }
}
```

GUI에서의 표시 변환:
```rust
fn format_timestamp(ts_str: &str) -> String {
    if let Ok(ts) = ts_str.parse::<u64>() {
        let days = ts / 86400;          // 초 → 일
        let years = days / 365;
        let remaining_days = days % 365;
        let months = remaining_days / 30;
        let day_of_month = (remaining_days % 30) + 1;
        format!("{:04}-{:02}-{:02}", 1970 + years, months + 1, day_of_month)
    } else {
        ts_str.to_string()
    }
}
```

⚠️ 이 변환은 윤년/월 길이를 무시하는 근사치. 정밀한 날짜가 필요하면 `chrono` crate 사용 권장.

## 정리: profile.rs의 함수 맵

```
save_profile(name, desc, profile) ──▶ 프로필 저장
    ├── profiles_dir()            ──▶ 저장 경로 결정
    ├── sanitize_profile_name()   ──▶ 파일명 안전화
    ├── MonitorProfile::new()     ──▶ 메타데이터 추가
    ├── serde_json::to_string_pretty() ──▶ JSON 변환
    └── fs::write()               ──▶ 파일 쓰기

load_profiles() ──▶ 전체 프로필 목록 로드
    ├── profiles_dir()
    ├── fs::read_dir()            ──▶ 디렉토리 순회
    ├── fs::read_to_string()      ──▶ 파일 읽기
    └── parse_profile_content()   ──▶ JSON 또는 레거시 파싱

load_profile_by_name(name) ──▶ 이름으로 프로필 찾기
    └── load_profiles() + find()

delete_profile(name) ──▶ 프로필 삭제
    └── fs::remove_file()
```

## 확인 문제

1. `serde_json::to_string_pretty`와 `serde_json::to_string`의 차이는?
2. `profiles_dir()`에서 `HOME`을 먼저 확인하고 `USERPROFILE`을 나중에 확인하는 이유는?
3. `sanitize_profile_name("집 모니터!")`의 결과는?
4. 레거시 프로필을 로드할 때 파일이 자동으로 새 포맷으로 업데이트되는가?

<details>
<summary>정답</summary>

1. `to_string_pretty`는 들여쓰기와 줄바꿈이 포함된 보기 좋은 JSON 생성. `to_string`은 공백 없는 한 줄 JSON. 파일 저장 시 사람이 읽을 수 있도록 pretty 사용.
2. Linux에서는 `HOME`이 표준이고 `USERPROFILE`은 없다. Windows에서는 `USERPROFILE`이 표준이지만 WSL 등에서 `HOME`도 설정되어 있을 수 있다. Linux 우선 지원이므로 `HOME`을 먼저 확인.
3. `"_________"` (9개의 언더스코어). 한글 3자("집", "모", "니"), 공백 1자, 한글 2자("터", "!") 모두 `_`로 대체. (정확히는 "집" "모니터!" 각 한글 문자 + 공백 + 느낌표 → 모두 `_`)
4. **아니다**. 로드 시 메모리에서 `MonitorProfile`로 변환하지만, 원본 파일은 수정하지 않는다. 다시 저장하면 새 포맷으로 저장됨.
</details>
