# DMaster로 배우는 Rust + Slint 커리큘럼

## 이 커리큘럼의 목표

DMaster 프로젝트의 실제 코드를 교재 삼아 세 가지를 익힌다:

1. **Rust스러운 설계** — Workspace, Module, Trait, Error Handling 등 관용적 패턴
2. **Core 로직의 흐름** — 디스플레이 조회 → 프로필 저장 → 적용까지의 전체 파이프라인
3. **Slint UI 수정 능력** — 직접 `.slint` 파일을 고쳐서 UI를 바꿀 수 있는 수준

---

## 프로젝트 전체 구조 (한눈에 보기)

```
DMaster/
├── Cargo.toml                 # Workspace root
├── dmaster_core/              # 핵심 라이브러리 (플랫폼 독립 로직)
│   └── src/
│       ├── lib.rs             # 모듈 선언 + re-export
│       ├── display_info.rs    # 데이터 모델 (struct/enum)
│       ├── query.rs           # 현재 디스플레이 조회
│       ├── apply.rs           # 프로필 적용
│       ├── profile.rs         # 프로필 저장/로드/삭제
│       └── backend/           # 플랫폼별 구현
│           ├── mod.rs         # DisplayBackend trait + get_backend()
│           ├── windows.rs     # Windows API (winapi crate)
│           ├── linux.rs       # X11 (xrandr)
│           ├── gnome_wayland.rs  # GNOME Wayland (Mutter D-Bus)
│           └── unsupported.rs # 미지원 플랫폼 폴백
├── dmaster_gui/               # GUI 프론트엔드 (Slint)
│   ├── build.rs               # Slint 컴파일 빌드 스크립트
│   ├── src/main.rs            # Rust ↔ Slint 바인딩
│   └── ui/
│       ├── app.slint          # 메인 윈도우
│       ├── save_dialog.slint  # 프로필 저장 다이얼로그
│       └── mapping_dialog.slint  # 디스플레이 매핑 다이얼로그
└── dmaster_cli/               # CLI 프론트엔드 (터미널 메뉴)
    └── src/main.rs
```
